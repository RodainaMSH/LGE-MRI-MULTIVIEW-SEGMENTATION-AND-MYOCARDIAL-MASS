## Critical Innovation: Voxel Volume Correction

### The Problem
**Previous approach (Run 2):**
- Hardcoded: 1.0 × 1.0 × 5.0 mm³ = 5.0 mm³
- Result: Up to 63% mass errors!

### The Solution
**Extract from DICOM metadata:**
- Tag (0028,0030): Pixel Spacing [row, col]
- Tag (0018,0050): Slice Thickness
- Calculate patient-specific voxel volume (mm³ → cm³)

### Impact

| Configuration | vPCC | RAE | Improvement |
|---------------|------|-----|-------------|
| Run 2: Hardcoded (1×1×5) | 0.8856 | 0.4820 | Baseline |
| **Run 6: Patient-specific** | **0.9077** | **0.3735** | **+2.5% vPCC** |
| | | | **-22.5% RAE** ✅ |

**Key Finding:** Simple preprocessing > Complex architecture

## Key Findings

### 1. Voxel Correction is CRITICAL
- **22.5% reduction** in mass error (RAE)
- MORE important than architectural complexity
- Essential for clinical deployment

### 2. Segmentation Dice ≠ Mass Accuracy

**Paradox:**
- Run 4: Higher Dice (0.8292) but Higher RAE (0.4153)
- **Run 6: Lower Dice (0.7560) but Lower RAE (0.3735)** ✅

**Explanation:**
- Mass depends on total pixels, not boundaries
- Voxel calibration affects ALL pixels equally
- Small boundary errors have minimal mass impact

### 3. Efficiency vs Performance

**Advantages over Attention U-Net:**
- ✅ Better mass accuracy (RAE 0.3735 vs 0.4153)
- ✅ Faster training (92 min vs 13 hours)
- ✅ Faster inference (100ms vs 150ms)
- ✅ Less memory (2.5GB vs 3.1GB)
- ✅ Simpler deployment

**Trade-offs:**
- ❌ Lower Dice (0.7560 vs 0.8292)
- ❌ Less sharp boundaries

### 4. Focal Loss Impact

**Minority class improvement:**
- Edema: +327% (0.12 → 0.51 Dice)
- Scar: +75% (0.35 → 0.61 Dice)
- Essential for class imbalance
